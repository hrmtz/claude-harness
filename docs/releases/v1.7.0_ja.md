# v1.7.0 — harness-kimi native hook 移植 (#54)

このリリースで harness-kimi を Kimi Code CLI の **native hook API** (Kimi >= 0.28
で提供) に移植し、長年の feature request だった
[#54](https://github.com/hrmtz/claude-harness/issues/54) を閉じました。
BASH_ENV 傍受レイヤー (#52) と wire watcher (#53) は deprecated です。
native PreToolUse が tool call 境界で同じ guard を発火させるため、傍受レイヤー
の既知バイパス (`bash --posix`, `bash -i`, `sh -c`) も構造的に閉じます。

## 変更内容

### native hooks が主層に

- リポジトリ直下に `install-kimi-hooks.sh` / `uninstall-kimi-hooks.sh` を新設
  (`install-grok-hooks.sh` の双子)。hook セットは `plugins/cross_cli_hooks.json`
  の kimi セクション、event/matcher/timeout は各 plugin の `hooks/hooks.json`
  (Claude/Codex/Grok と同じ SSOT) から引きます。生成した `[[hooks]]` TOML は
  `~/.kimi-code/config.toml` 内のマーカーブロックに冪等マージされ、
  `~/sanada_backup_persistent/` への backup と `tomllib` 検証を経てから
  書き込まれます (Kimi は `[[hooks]]` の未知フィールドで config 全体を拒否
  するため)。
- overlay の kimi セクションを `insurance/gates/hints` 形式から grok 型の
  `hooks` リストに再構成。15 hooks: Bash gate 群 (bash_command_guard,
  branch_policy, pg_rotation_propagation, pipeline preflight, phase review,
  sanada autobackup, long_task advisor)、Read / MCP の credential file read
  guard、Write|Edit チェック、PostToolUse credential scrub、UserPromptSubmit
  reminder。

### Kimi 0.28.1 での実測 (2026-07-21)

- hook ペイロードは Claude 形状の snake_case (`tool_name`,
  `tool_input.command`, `.prompt`, `session_id`, `cwd`, `tool_call_id`)。
- Kimi は Claude の JSON contract を **余分な `hookEventName` フィールド込みで**
  受理します: `hookSpecificOutput.permissionDecision: "deny"` で block、
  `hookSpecificOutput.additionalContext` で UserPromptSubmit への context 注入。
  したがって harness-core の hook スクリプトは **無改変で動きます**。
- 2つの差分は `plugins/harness-core/hooks/lib.sh` が吸収: PostToolUse の結果は
  `.tool_response` ではなく `.tool_output` の文字列、`transcript_path` は無く
  session log は `$KIMI_CODE_HOME/sessions/<wd>/<session_id>/agents/main/wire.jsonl`
  (ペイロードの `session_id` から解決可能)。`active_jsonl` /
  `parse_tool_output` / `recent_assistant_turns` に Kimi ブランチを追加
  (Claude/Codex/Grok の挙動は不変。既存テストスイート全パス)。

### 旧レイヤーの退役

- `kimi-wrapper.sh` は `BASH_ENV` を export しなくなり、
  `HARNESS_KIMI_BASH_GUARD` は無視されます。guard/watcher スクリプトは
  uninstall 経路を断たないため repo に残します (deprecated)。移行手順は
  `plugins/harness-kimi/README.md` を参照。
- wire.jsonl の cron scrubber は **維持**: Kimi の PostToolUse は observe-only
  (block 不可) のため、定期的な detective 層は依然として必須です。
- `scripts/check_cross_cli_hooks.sh --live` は `config.toml` の kimi マーカー
  ブロックと overlay の一致を検査します。

### 未移植

- `sr_depth_gate.py` と `stall_autocontinue.sh` (Stop/SubagentStop) は Claude
  transcript 構造に依存するため Claude/Codex 専用のままです。

## 検証

- `plugins/harness-kimi/tests/test_kimi_native_hooks.py` 新規 (6 tests):
  deny JSON 形状、良性コマンド素通り、`.tool_output` パース、wire.jsonl 解決、
  installer の冪等性/設定保持、uninstaller。
- Kimi 0.28.1 での E2E (`kimi -p`): 禁止コマンド `sops -d` が block され理由が
  モデルに到達、良性コマンドは素通り、合成 credential の echo は wire.jsonl の
  tool result からリアルタイムに redact。
- `scripts/check_cross_cli_hooks.sh --live` の kimi セクション: 同期。

## アップグレード

```bash
git pull
bash install-kimi-hooks.sh                       # native hooks を配線
bash plugins/harness-kimi/uninstall-kimi-watcher.sh   # 旧 watcher cron がある場合
rm -rf ~/.kimi-code/bin/guarded-bash-dir               # 旧 BASH_ENV guard がある場合
bash plugins/harness-kimi/install-kimi-wrapper.sh      # wrapper 更新 (BASH_ENV 除去)
```

Kimi Code CLI >= 0.28 と、installer 用に python3 >= 3.11 (tomllib) が必要です。
