# v1.7.0 — harness-kimi native hook port (#54)

This release ports harness-kimi to Kimi Code CLI's **native hook API** (shipped
in Kimi >= 0.28), closing the long-standing feature request in
[#54](https://github.com/hrmtz/claude-harness/issues/54). The BASH_ENV
interception layer (#52) and the wire watcher (#53) are deprecated — the native
PreToolUse event fires the same guards at the tool-call boundary, which also
closes the interception layer's documented bypasses (`bash --posix`, `bash -i`,
`sh -c`).

## What changed

### Native hooks are the primary layer

- New `install-kimi-hooks.sh` / `uninstall-kimi-hooks.sh` at the repo root —
  close cousins of `install-grok-hooks.sh`. The hook set comes from the kimi
  section of `plugins/cross_cli_hooks.json`; event/matcher/timeout are looked
  up from each plugin's `hooks/hooks.json` (the same SSOT that drives Claude,
  Codex, and Grok). The generated `[[hooks]]` TOML is merged into
  `~/.kimi-code/config.toml` inside a marker block — idempotent, backed up to
  `~/sanada_backup_persistent/`, and validated with `tomllib` before install
  (Kimi rejects the whole config on unknown `[[hooks]]` fields).
- The kimi overlay section is restructured from the `insurance/gates/hints`
  shape to the grok-style `hooks` list — now 15 hooks: the Bash gates
  (bash_command_guard, branch_policy, pg_rotation_propagation, pipeline
  preflight, phase review, sanada autobackup, long_task advisor), the Read /
  MCP credential file read guard, the Write|Edit checks, the PostToolUse
  credential scrubs, and the UserPromptSubmit reminders.

### Empirically verified against Kimi 0.28.1 (2026-07-21)

- The hook payload is Claude-shaped snake_case (`tool_name`,
  `tool_input.command`, `.prompt`, `session_id`, `cwd`, `tool_call_id`).
- Kimi honors the Claude JSON contract **including the extra `hookEventName`
  field**: `hookSpecificOutput.permissionDecision: "deny"` blocks, and
  `hookSpecificOutput.additionalContext` injects context on UserPromptSubmit.
  The harness-core hook scripts therefore run **unmodified**.
- Two deltas are absorbed by `plugins/harness-core/hooks/lib.sh`:
  PostToolUse carries the result as a plain string under `.tool_output`
  (not `.tool_response`), and there is no `transcript_path` — the session log
  is `$KIMI_CODE_HOME/sessions/<wd>/<session_id>/agents/main/wire.jsonl`,
  resolvable from the payload's `session_id`. `active_jsonl`,
  `parse_tool_output`, and `recent_assistant_turns` gained Kimi branches
  (Claude/Codex/Grok behavior unchanged; existing test suite passes).

### Legacy layer retirement

- `kimi-wrapper.sh` no longer exports `BASH_ENV`; `HARNESS_KIMI_BASH_GUARD` is
  ignored. The guard/watcher scripts stay in the repo (deprecated) so existing
  installs can be uninstalled — migration steps are in
  `plugins/harness-kimi/README.md`.
- The wire.jsonl cron scrubber **remains**: PostToolUse is observe-only on
  Kimi (it cannot block), so the periodic detective layer is still load-bearing.
- `scripts/check_cross_cli_hooks.sh --live` now checks the kimi marker block
  in `config.toml` against the overlay.

### Not ported

- `sr_depth_gate.py` and `stall_autocontinue.sh` (Stop/SubagentStop) parse
  Claude transcript structure and remain Claude/Codex-only.

## Verification

- New `plugins/harness-kimi/tests/test_kimi_native_hooks.py` (6 tests):
  deny JSON shape, benign pass-through, `.tool_output` parsing, wire.jsonl
  resolution, installer idempotency/settings preservation, uninstaller.
- E2E on Kimi 0.28.1 (`kimi -p`): a forbidden `sops -d` command is blocked
  with the guard's reason reaching the model; benign commands pass; a
  synthetic credential echoed to stdout is redacted from the wire.jsonl
  tool result in real time.
- `scripts/check_cross_cli_hooks.sh --live` kimi section: in sync.

## Upgrade

```bash
git pull
bash install-kimi-hooks.sh                       # wire native hooks
bash plugins/harness-kimi/uninstall-kimi-watcher.sh   # if the old watcher cron exists
rm -rf ~/.kimi-code/bin/guarded-bash-dir               # if the old BASH_ENV guard exists
bash plugins/harness-kimi/install-kimi-wrapper.sh      # refresh the wrapper (drops BASH_ENV)
```

Requires Kimi Code CLI >= 0.28 and python3 >= 3.11 (tomllib) for the installer.
